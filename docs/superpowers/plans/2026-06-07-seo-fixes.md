## Trailing Slash — How It Actually Works

CF Pages handles trailing slash **redirects** (actual HTTP requests) via `public/_redirects`. That part is correct and working.

The bug in `BreadcrumbSchema.astro` is different: `withSlash()` adds trailing slashes to URLs **inside the JSON-LD script tag** — these are just strings in HTML, never pass through CF redirects. Google reads these strings directly and they must match the canonical URL exactly (no trailing slash). Fix = remove `withSlash()` from schema generation only. Visual breadcrumb links (`href={item.url}` at line 44) are already correct.
